Правила оформления домашних заданий

JavaScript v2.0

  1. Все домашние задания выполняются и принимаются к проверке только в том случае, если выполнены на основе шаблона.
  2. Шаблон можно скачать с помощью кнопки под видео.
  3. Шаблон - инструмент обучения. Изучите его стиль, синтаксис, способы именования и вызова переменных, функций. Задача шаблона - выработать у вас правильный стиль программирования и написания программ.
  4. Запрещается переименовывать файлы шаблона, изменять способы объявления и имена созданных переменных, функций, способы вызова функций. При невыполнении этих правил - работа будет возвращена.
  5. Учащийся делает ДЗ прямо в шаблоне, после чего запаковывает архив - и загружает на странице юнита.
  6. Вы можете до получения оценки загружать ДЗ любое количество раз, при этом предыдущий файл будет заменен.
  7. Размер загружаемого файла не может превышать 1мб.
  8. После загрузки ваша работа видна в кабинете под именем unit.zip и оценкой 0 баллов. Это значит, что работа ожидает проверки.
  9. В ходе проверки оценка в любом случае изменится и будет больше 0 баллов.
  10. Если вы получили не максимальную оценку - исправьте работу и загрузите на перепроверку. Для перепроверки вы также загружаете всю работу целиком, даже если изменяли одну задачу.
  11. Загруженная работа получает название fix.
  12. Проверка fix - финальная проверка данного юнита. После этого данный юнит проверятся не будет.

React Lite Level

  1. После выполнения ДЗ вы создаете zip архив. Удаляете в нем папки node_modules и package-lock.json. Размер ДЗ не должен превышать 1Мб.
  2. Не устанавливайте сторонних пакетов, которые не использовались в уроке.
  3. Вы можете до получения оценки загружать ДЗ любое количество раз, при этом предыдущий файл будет заменен.
  4. После загрузки ваша работа видна в кабинете под именем unit.zip и оценкой 0 баллов. Это значит, что работа ожидает проверки.
  5. В ходе проверки оценка в любом случае изменится и будет больше 0 баллов.
  6. Если вы получили не максимальную оценку - исправьте работу и загрузите на перепроверку. Для перепроверки вы также загружаете всю работу целиком, даже если изменяли одну задачу.
  7. Загруженная работа получает название fix.
  8. Проверка fix - финальная проверка данного юнита. После этого данный юнит проверятся не будет.
  9. Работы с 15 до 18 это выполнение проекта. Не нужно их загружать, вы загружаете только 19 юнит и ожидаете проверки.
